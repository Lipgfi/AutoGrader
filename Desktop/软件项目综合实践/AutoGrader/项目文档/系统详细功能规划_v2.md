# AutoGrader 系统详细功能规划

---

## 一、用户角色与权限体系

系统设三种角色，权限逐级递增：


| 角色              | 说明                  | 典型操作                  |
| --------------- | ------------------- | --------------------- |
| **Student（学生）** | 通过注册或导入创建，绑定班级与课程   | 查看课程、提交作业、查看成绩        |
| **Teacher（教师）** | 由管理员创建账号，管理自己的课程与班级 | 建课、建班、导入学生、发布作业、查成绩   |
| **Admin（管理员）**  | 最高权限，负责初始化与系统级管理    | 创建教师账号、批量导入学生、查看全平台数据 |


---

## 二、学生端（Student Portal）

### 2.1 注册与登录

#### 注册页面

学生可以**自行注册**，也可以由教师导入后**激活账号**（两种方式都要支持）。

**注册表单字段：**


| 字段                 | 类型     | 是否必填 | 规则                            |
| ------------------ | ------ | ---- | ----------------------------- |
| `username`         | string | 是    | 唯一，4~20位字母数字，注册后**不可修改**      |
| `password`         | string | 是    | ≥8位，含字母+数字                    |
| `confirm_password` | string | 是    | 与 password 一致，仅前端校验           |
| `real_name`        | string | 是    | 真实姓名，注册后**不可自行修改**（由教师/管理员修改） |
| `student_id`       | string | 是    | 学号，唯一，注册后**不可修改**             |
| `email`            | string | 是    | 唯一，用于找回密码                     |
| `phone`            | string | 否    | 手机号                           |


> **说明**：学号（`student_id`）注册后不可修改，因为它是学生在学校的唯一身份标识，绑定成绩等核心数据；若填错需联系教师/管理员处理。

**注册后流程：**

- 自行注册的学生，账号默认无班级，需等待教师将其加入班级；
- 教师通过导入名单创建的学生，首次登录时需修改初始密码。

---

#### 登录页面


| 字段                   | 说明           |
| -------------------- | ------------ |
| `username` 或 `email` | 两者均可登录       |
| `password`           | 密码           |
| 记住我                  | 7天 Token 有效期 |
| 忘记密码                 | 通过邮箱验证码重置    |

**可查看的字段（全部）：**


| 字段           | 显示名    | 是否可自行修改 | 备注       |
| ------------ | ------ | ------- | -------- |
| `username`   | 用户名    | ❌ 不可改   | 唯一标识     |
| `student_id` | 学号     | ❌ 不可改   | 由学校/导入决定 |
| `real_name`  | 真实姓名   | ❌ 不可改   | 需联系教师修改  |
| `email`      | 邮箱     | ✅ 可改    | 改后需邮箱验证  |
| `phone`      | 手机号    | ✅ 可改    | 可选       |
| `avatar`     | 头像     | ✅ 可改    | 上传图片     |

| `role`       | 角色     | ❌ 不可改   | 显示"学生"   |
| `created_at` | 注册时间   | ❌ 只读    | 系统自动     |
| `classes`    | 所在班级列表 | ❌ 只读    | 由教师分配    |

### 2.3 课程与班级

#### 我的课程页面

学生可以看到自己**被加入的所有课程**，每张课程卡片显示：


| 显示内容       | 对应字段                                     |
| ---------- | ---------------------------------------- |
| 课程名称       | `course_name`                            |
| 课程编号       | `course_code`                            |
| 所在班级       | `class_name`                             |
| 授课教师       | `teacher_name`                           |
| 学期         | `semester`                               |
| 未完成作业数（角标） | 统计 `assignments` 中 `due_date` 未过期且未提交的数量 |


点击课程卡片进入**课程详情页**，可查看：

- 课程公告（教师发布）
- 该课程下的全部作业列表
- 班级成员（同班同学姓名列表，不显示成绩）

---

### 2.4 作业与答题

#### 作业列表页

每条作业显示：


| 显示内容 | 字段                  | 说明                  |
| ---- | ------------------- | ------------------- |
| 作业标题 | `assignment_title`  |                     |
| 截止时间 | `due_date`          | 过期显示红色              |
| 完成状态 | 已提交/未提交/部分提交        | 根据 `submissions` 计算 |
| 我的得分 | `my_score`          | 显示“该次作业（即该题）的最终得分”          |

删除：“题目数量 | 统计 questions 列表长度”这一行。
修改：“我的得分”不再是“汇总该作业下各题得分”，而是直接显示“该次作业（即该题）的最终得分”。

#### 答题页面（核心页面）

进入单次作业后：

**左侧：** 题目描述区

- 题目标题
- 题目描述（支持 Markdown 渲染）
- 输入/输出示例（来自 B-3 的示例测试用例）
- 难度等级、时间限制、内存限制

**右侧：** 代码编辑区

- 代码编辑器（支持语法高亮，如 Monaco Editor）
- 语言选择（先完成Python、后续加入对C/C++、Java的支持）
- 提交按钮
- 历史提交记录（可查看之前的提交代码和得分）

**提交流程：**

1. 学生点击"提交"
2. B-1 将代码 POST 到 B-2 的提交接口
3. 页面进入"评测中"状态（轮询或 WebSocket 等待结果）
4. 结果返回后展示评分详情

### 2.5 成绩查看

#### 成绩总览页


| 维度  | 内容                   |
| --- | -------------------- |
| 按课程 | 每门课程的总分（各作业得分加权/求和）  |
| 按作业 | 该作业下得分、班级排名（可选） |

改了哪里：去掉了“各题得分”的概念。
怎么改：“按作业”维度的内容修改为：“该作业得分、班级排名（可选）”。

#### 单题评分详情页


| 显示内容    | 字段                                 |
| ------- | ---------------------------------- |
| 总体评价    | `overall_comment`（如"通过 4/5 个测试用例"） |
| 得分      | `overall_score`                    |
| 静态问题    | `static_issues`（如"使用了禁止的模块 os"）    |
| 各用例详情   | `case_results` 列表                  |
| └─ 用例编号 | `case_id`                          |
| └─ 是否通过 | `passed`（通过显示绿色✓，失败显示红色✗）          |
| └─ 期望输出 | `expected_output`                  |
| └─ 实际输出 | `actual_output`                    |
| └─ 运行时间 | `execution_time_ms`                |
| └─ 错误信息 | `error`（如"运行超时"）                   |

## 三、教师端（Teacher Portal）

### 3.1 登录

教师账号由管理员创建，不允许自行注册（防止非授权人员冒充教师）。

登录字段与学生端相同，登录后跳转到教师工作台。

---

### 3.2 个人信息


| 字段           | 显示名  | 是否可自行修改      |
| ------------ | ---- | ------------ |
| `username`   | 用户名  | ❌ 不可改        |
| `teacher_id` | 工号   | ❌ 不可改        |
| `real_name`  | 真实姓名 | ❌ 不可改（管理员修改） |
| `email`      | 邮箱   | ✅ 可改         |
| `phone`      | 手机号  | ✅ 可改         |
| `department` | 院系   | ✅ 可改         |
| `avatar`     | 头像   | ✅ 可改         |
| `password`   | 密码   | ✅ 可改         |


---

### 3.3 课程管理

#### 课程列表页

教师可看到自己创建的全部课程，以及每门课的班级数、学生总数、作业数。

#### 新建课程


| 字段            | 类型     | 是否必填 | 说明                  |
| ------------- | ------ | ---- | ------------------- |
| `course_name` | string | 是    | 课程名称，如"Python 程序设计" |
| `course_code` | string | 是    | 课程编号，如"CS102"，同学期唯一 |
| `semester`    | string | 是    | 学期，如"2025-2026-2"   |
| `description` | text   | 否    | 课程简介                |


创建课程后，教师可在课程下创建多个**班级（课序号）**。

---

### 3.4 班级管理

一门课程可以有多个班级（对应"课序号"概念），例如"CS102-01班"、"CS102-02班"。

#### 新建班级


| 字段           | 类型     | 说明             |
| ------------ | ------ | -------------- |
| `class_name` | string | 班级名称，如"软件工程2班" |
| `class_code` | string | 课序号，如"01"      |
| `course_id`  | FK     | 所属课程，自动关联      |


#### 班级详情页

- 显示班级学生列表
- 支持：搜索学生、添加学生、移除学生
- 显示该班级的平均分、最高/最低分（成绩概览）

---

### 3.5 学生管理

#### 导入学生名单（批量导入）

支持上传 **PDF** 或 **XLSX** 格式文件，系统解析后预览，确认后批量创建/关联学生账号。

**文件中必须包含的字段：**


| 字段            | 说明     | 处理逻辑           |
| ------------- | ------ | -------------- |
| `real_name`   | 真实姓名   | 直接写入           |
| `student_id`  | 学号     | 全局唯一，若已存在则直接关联 |
| `course_code` | 课程编号   | 用于匹配已有课程       |
| `class_code`  | 课序号    | 用于匹配/创建班级      |
| `email`       | 邮箱（可选） | 若有则发送初始密码邮件    |


**导入逻辑：**

- 若 `student_id` 对应账号已存在：直接将该学生加入对应班级；
- 若不存在：自动创建账号，初始密码为学号，首次登录强制修改；
- 导入结果展示：成功 N 条，失败 M 条（附失败原因，如"学号已存在于其他班级"）。

#### 手动添加单个学生

通过学号搜索已有账号，直接加入班级。

---

### 3.6 题库管理（对接 B-3）

> 教师需要在题库中维护题目，这是 B-1 教师端调用 B-3 API 的核心场景。

#### 题目列表页

显示教师创建的全部题目，支持按**题型**（CLI/文件/接口）、**难度**、**课程**筛选。

#### 新建题目

**基本信息（必填）：**


| 字段                | 类型       | 说明                        |
| ----------------- | -------- | ------------------------- |
| `title`           | string   | 题目标题                      |
| `description`     | markdown | 题干描述（支持 Markdown）         |
| `question_type`   | enum     | 1=命令行输入 / 2=文件读取 / 3=接口调用 |
| `difficulty`      | enum     | EASY / MEDIUM / HARD      |
| `time_limit_ms`   | int      | 运行超时（ms），默认 3000          |
| `memory_limit_mb` | int      | 内存限制（MB），默认 64            |
| `language`        | string   | 目标语言，当前 "python"          |


**题型专用配置（按类型显示不同表单）：**

- **CLI 题**：无额外配置（stdin 自动注入）
- **文件题**：需填 `input_filename`（如 `input.txt`），可选 `output_filename`
- **接口题**：需填 `entry_function`（函数名，如 `is_palindrome`），可选 `setup_code`（调用前准备代码）

**测试用例（可添加多条）：**


| 字段                | 说明                               |
| ----------------- | -------------------------------- |
| `case_id`         | 自动生成，如 case_01                   |
| `description`     | 用例描述，如"包含负数"                     |
| `input_data`      | CLI/文件题的输入内容                     |
| `call_args`       | 接口题的调用参数，JSON 数组，如 `["racecar"]` |
| `expected_output` | 期望输出字符串                          |
| `score_weight`    | 权重，默认 1.0                        |


**标准答案代码（可选但强烈建议填写）：**

- 填写后点击"验证答案"，B-1 调用 B-3 的 `/api/v1/b3/evaluate/answer/{id}` 接口，展示预评分结果；
- 若标答未通过所有用例，系统警告但仍允许保存。

### 3.7 作业管理

#### 新建作业

| `allow_resubmit`   | bool     | 是否允许重复提交，默认允许 |


**发布后不可修改的字段：** `question_id`（题目列表不能改，防止已有学生提交与题目不对应）

**发布后可修改的字段：** `due_date`（可延期）、`description`（可补充说明）

改了哪里：作业与题库的绑定关系从“一对多（列表）”变成了“一对一”。
怎么改：
将 questions（数组）字段修改为 question_id（单值）。

#### 作业详情页

- 查看该作业的**全班提交情况**：提交人数/总人数、各题通过率；
- 可查看单个学生的提交代码与评价结果；
- 支持**导出成绩表**（XLSX 格式，含学号、姓名、作业得分）。

改了哪里：去掉了分题统计的逻辑。
怎么改：
将“查看该作业各题的全班提交情况”修改为“查看该作业的全班提交情况（提交人数/总人数、通过率）”。
导出成绩表的说明，将“含学号、姓名、各题得分、总分”修改为“含学号、姓名、作业得分”。

## 四、管理员端（Admin Portal）

管理员端可以做成轻量的后台管理页面（如 `/admin` 路由），不需要独立 App。

### 4.1 用户管理

#### 教师账号管理


| 操作      | 说明                     |
| ------- | ---------------------- |
| 创建教师账号  | 填写工号、姓名、邮箱、初始密码，发送激活邮件 |
| 停用/恢复账号 | 停用后教师无法登录，数据保留         |
| 重置密码    | 管理员可为任何用户重置密码          |
| 查看教师列表  | 工号、姓名、邮箱、创建时间、所管课程数    |


#### 学生账号管理


| 操作     | 说明                   |

| 修改学生信息 | 可修改真实姓名、学号（谨慎操作）     |

#### 创建教师账号字段


| 字段                 | 类型     | 说明             |
| ------------------ | ------ | -------------- |

| `department`       | string | 院系（可选）         |
| `initial_password` | string | 初始密码，首次登录强制修改  |


### 4.2 系统管理

- 查看全平台统计（总用户数、总课程数、总提交数）
- 查看系统日志（可选，记录异常操作）
- 查看各模块运行状态（B-2/B-3/B-4 的健康检查接口）

## 五、核心数据模型（字段完整定义）

### 5.1 用户表 `users`


| 字段              | 类型          | 约束               | 说明                              |
| --------------- | ----------- | ---------------- | ------------------------------- |
| `user_id`       | UUID/int    | PK, 自增           | 系统内部ID                          |
| `username`      | varchar(20) | UNIQUE, NOT NULL | 登录用户名                           |
| `password_hash` | varchar     | NOT NULL         | bcrypt 加密存储                     |
| `email`         | varchar     | UNIQUE, NOT NULL | 邮箱                              |
| `phone`         | varchar     | NULLABLE         | 手机号                             |
| `real_name`     | varchar     | NOT NULL         | 真实姓名                            |
| `role`          | enum        | NOT NULL         | `student` / `teacher` / `admin` |
| `avatar_url`    | varchar     | NULLABLE         | 头像地址                            |
| `is_active`     | bool        | DEFAULT true     | 是否启用                            |
| `created_at`    | datetime    | AUTO             | 注册/创建时间                         |
| `last_login_at` | datetime    | NULLABLE         | 最后登录时间                          |


### 5.2 学生扩展表 `students`


| 字段                       | 类型         | 约束               | 说明          |
| ------------------------ | ---------- | ---------------- | ----------- |
| `user_id`                | FK → users | PK               | 与 users 1:1 |
| `student_id`             | varchar    | UNIQUE, NOT NULL | 学号          |
| `first_password_changed` | bool       | DEFAULT false    | 是否已修改初始密码   |


### 5.3 教师扩展表 `teachers`


| 字段           | 类型         | 约束               | 说明          |
| ------------ | ---------- | ---------------- | ----------- |
| `user_id`    | FK → users | PK               | 与 users 1:1 |
| `teacher_id` | varchar    | UNIQUE, NOT NULL | 工号          |
| `department` | varchar    | NULLABLE         | 院系          |


### 5.4 课程表 `courses`


| 字段            | 类型         | 约束       | 说明               |
| ------------- | ---------- | -------- | ---------------- |
| `course_id`   | int        | PK, 自增   |                  |
| `course_code` | varchar    | NOT NULL | 课程编号，如 CS102     |
| `course_name` | varchar    | NOT NULL | 课程名称             |
| `teacher_id`  | FK → users | NOT NULL | 创建该课程的教师         |
| `semester`    | varchar    | NOT NULL | 学期，如 2025-2026-2 |
| `description` | text       | NULLABLE | 课程简介             |
| `created_at`  | datetime   | AUTO     |                  |


### 5.5 班级表 `classes`


| 字段           | 类型           | 约束       | 说明       |
| ------------ | ------------ | -------- | -------- |
| `class_id`   | int          | PK, 自增   |          |
| `course_id`  | FK → courses | NOT NULL | 所属课程     |
| `class_name` | varchar      | NOT NULL | 班级名称     |
| `class_code` | varchar      | NOT NULL | 课序号，如 01 |
| `teacher_id` | FK → users   | NOT NULL | 管理该班的教师  |
| `created_at` | datetime     | AUTO     |          |


### 5.6 学生-班级关联表 `class_students`


| 字段                | 类型           | 约束       |
| ----------------- | ------------ | -------- |
| `class_id`        | FK → classes | PK（联合主键） |
| `student_user_id` | FK → users   | PK（联合主键） |
| `joined_at`       | datetime     | AUTO     |


### 5.7 作业表 `assignments`

## 六、模块间 API 规范

### API 版本前缀（v1）

为便于联调与后续兼容，**所有对外 HTTP 接口统一挂载在 `/api/v1` 下**：

- **B-4、B-2**：路径形如 `/api/v1/auth/...`、`/api/v1/submissions/...` 等。
- **B-3**：在版本前缀下保留模块标识，形如 `/api/v1/b3/questions`、`/api/v1/b3/evaluate` 等。

破坏性变更应通过新版本（如 `v2`）发布；`v1` 在验收周期内保持稳定。
