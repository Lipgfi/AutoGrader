# 模块之间的接口设计

## 一、前端组件间接口

### 1.1 组件通信方式

#### 1.1.1 父子组件通信

**Props 传递**：
- **Direction**: 父 → 子
- **Usage**: 父组件通过 props 向子组件传递数据和回调函数
- **Example**:
  ```typescript
  // 父组件
  <CodeEditor
    :code="currentCode"
    :language="selectedLanguage"
    @update:code="handleCodeUpdate"
    @run-test="handleRunTest"
  />
  
  // 子组件
  defineProps<{
    code: string
    language: string
  }>()
  
  const emit = defineEmits<{
    'update:code': [value: string]
    'run-test': []
  }>()
  ```

**事件传递**：
- **Direction**: 子 → 父
- **Usage**: 子组件通过 emit 触发事件，父组件监听事件并响应
- **Example**:
  ```typescript
  // 子组件
  const handleSubmit = () => {
    emit('submit', code.value)
  }
  
  // 父组件
  const handleSubmit = (code: string) => {
    // 处理提交逻辑
  }
  ```

#### 1.1.2 兄弟组件通信

**事件总线**：
- **Direction**: 双向
- **Usage**: 通过全局事件总线实现兄弟组件通信
- **Example**:
  ```typescript
  // eventBus.ts
  import mitt from 'mitt'
  export const eventBus = mitt()
  
  // 组件 A
  eventBus.emit('updateAssignment', assignment)
  
  // 组件 B
  eventBus.on('updateAssignment', (assignment) => {
    // 处理更新
  })
  ```

**Pinia 状态管理**：
- **Direction**: 双向
- **Usage**: 通过共享状态实现兄弟组件通信
- **Example**:
  ```typescript
  // store/assignment.ts
  export const useAssignmentStore = defineStore('assignment', {
    state: () => ({
      currentAssignment: null
    }),
    actions: {
      setCurrentAssignment(assignment) {
        this.currentAssignment = assignment
      }
    }
  })
  
  // 组件 A
  const assignmentStore = useAssignmentStore()
  assignmentStore.setCurrentAssignment(assignment)
  
  // 组件 B
  const assignmentStore = useAssignmentStore()
  watch(() => assignmentStore.currentAssignment, (newAssignment) => {
    // 处理更新
  })
  ```

#### 1.1.3 跨层级组件通信

**Provide/Inject**：
- **Direction**: 祖先 → 后代
- **Usage**: 祖先组件 provide 数据，后代组件 inject 使用
- **Example**:
  ```typescript
  // 祖先组件
  provide('userInfo', userInfo)
  
  // 后代组件
  const userInfo = inject('userInfo')
  ```

**Pinia 状态管理**：
- **Direction**: 全局
- **Usage**: 通过全局状态管理实现跨层级通信
- **Example**:
  ```typescript
  // store/user.ts
  export const useUserStore = defineStore('user', {
    state: () => ({
      userInfo: null
    }),
    actions: {
      setUserInfo(info) {
        this.userInfo = info
      }
    }
  })
  
  // 任何组件
  const userStore = useUserStore()
  userStore.setUserInfo(info)
  ```

### 1.2 关键组件接口

#### 1.2.1 登录组件 (Login.vue)

**Props**：
- `defaultRole` (string): 默认角色选择，可选值：'student', 'teacher', 'admin'
- `redirectPath` (string): 登录成功后重定向路径

**Events**：
- `login-success`: 登录成功时触发，参数：`{token, userRole, userInfo}`
- `login-failure`: 登录失败时触发，参数：`{error}`

**Methods**：
- `validateForm()`: 验证表单，返回布尔值
- `submitLogin()`: 提交登录请求

#### 1.2.2 课程列表组件 (MyCourses.vue)

**Props**：
- `studentId` (string): 学生ID
- `semester` (string): 学期筛选
- `viewMode` (string): 视图模式，可选值：'card', 'list'

**Events**：
- `course-selected`: 选择课程时触发，参数：`courseId`
- `view-changed`: 视图模式改变时触发，参数：`viewMode`

**Methods**：
- `loadCourses()`: 加载课程列表
- `filterCourses()`: 筛选课程
- `switchView()`: 切换视图模式

#### 1.2.3 在线编程组件 (CodingPage.vue)

**Props**：
- `assignmentId` (string): 作业ID
- `questionId` (string): 题目ID
- `initialCode` (string): 初始代码
- `language` (string): 编程语言

**Events**：
- `code-updated`: 代码更新时触发，参数：`code`
- `submission-success`: 提交成功时触发，参数：`result`
- `submission-failure`: 提交失败时触发，参数：`error`

**Methods**：
- `loadQuestion()`: 加载题目信息
- `runTest()`: 运行测试用例
- `submitCode()`: 提交代码
- `saveDraft()`: 保存草稿

#### 1.2.4 成绩管理组件 (Grades.vue)

**Props**：
- `studentId` (string): 学生ID
- `courseId` (string): 课程ID
- `viewMode` (string): 视图模式，可选值：'course', 'assignment'

**Events**：
- `grade-selected`: 选择成绩时触发，参数：`{assignmentId, submissionId}`
- `view-changed`: 视图模式改变时触发，参数：`viewMode`

**Methods**：
- `loadGrades()`: 加载成绩数据
- `filterGrades()`: 筛选成绩
- `switchView()`: 切换视图模式

## 二、前后端接口

### 2.1 API 基础配置

**基础 URL**：`/api`

**请求头**：
- `Content-Type`: `application/json`
- `Authorization`: `Bearer {token}` (登录后)

**响应格式**：
```json
{
  "code": 200, // 状态码
  "message": "success", // 提示信息
  "data": {} // 数据
}
```

### 2.2 认证相关接口

#### 2.2.1 登录接口

**路径**：`/api/auth/login`
**方法**：`POST`
**请求体**：
```json
{
  "username": "string", // 用户名或邮箱
  "password": "string", // 密码
  "role": "string" // 角色：student/teacher/admin
}
```
**响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "string", // JWT令牌
    "userInfo": {
      "userId": "string",
      "username": "string",
      "realName": "string",
      "role": "string"
    }
  }
}
```

#### 2.2.2 注册接口

**路径**：`/api/auth/register`
**方法**：`POST`
**请求体**：
```json
{
  "username": "string", // 用户名
  "password": "string", // 密码
  "realName": "string", // 真实姓名
  "studentId": "string", // 学号
  "email": "string", // 邮箱
  "phone": "string" // 手机号
}
```
**响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": "string"
  }
}
```

#### 2.2.3 登出接口

**路径**：`/api/auth/logout`
**方法**：`POST`
**响应**：
```json

  "code": 200,
  "message": "success"
}
```

## 三、数据流设计

### 3.1 前端数据流

**用户认证数据流**：
```
用户输入 → 表单验证 → API请求 → 响应处理 → 本地存储Token → 状态更新 → 路由跳转
```

**课程管理数据流**：
```
页面加载 → API请求课程列表 → 数据处理 → 状态更新 → 渲染课程列表 → 用户选择 → 加载课程详情
```

**在线编程数据流**：
```
页面加载 → 加载题目信息 → 初始化编辑器 → 代码编辑 → 本地存储 → 提交代码 → API请求评测 → 显示评测结果
```

**成绩管理数据流**：
```
页面加载 → API请求成绩数据 → 数据处理 → 状态更新 → 渲染成绩列表 → 用户选择 → 加载成绩详情
```

### 3.2 后端数据流

**认证数据流**：
```
接收请求 → 验证参数 → 数据库查询 → 密码验证 → 生成Token → 返回响应
```

**评测数据流**：
```
接收提交 → 验证参数 → 代码存储 → 启动评测 → 执行测试用例 → 收集结果 → 计算分数 → 存储结果 → 返回响应
```

**成绩数据流**：
```
接收查询 → 验证参数 → 数据库查询 → 数据聚合 → 计算统计 → 返回响应
```

### 3.3 数据同步策略

**实时同步**：
- 用户认证状态
- 提交结果
- 评测状态

**按需同步**：
- 课程列表
- 作业列表
- 题目列表
- 成绩数据

**缓存策略**：
- 本地存储：用户信息、Token、草稿代码
- 内存缓存：课程列表、作业列表
- 服务器缓存：热点数据（如题目信息）

### 3.4 错误处理机制

**前端错误处理**：
- API请求错误：统一拦截器处理
- 表单验证错误：实时反馈
- 运行时错误：全局错误处理

**后端错误处理**：
- 参数验证错误：返回400状态码
- 认证错误：返回401状态码
- 权限错误：返回403状态码
- 服务器错误：返回500状态码

**错误日志**：
- 前端：控制台日志 + 错误上报
- 后端：服务器日志 + 数据库存储
